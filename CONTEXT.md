# 📋 TaskBoard — Контекст и Документация Проекта

Данный документ является единым источником правды (Single Source of Truth) по архитектуре, функционалу и текущему состоянию проекта **TaskBoard**.

---

## 🛠 1. Стек Технологий и Архитектура

- **Фреймворк**: Next.js (App Router, JavaScript, CSS Variables без Tailwind).
- **База Данных**: Supabase (изолированные таблицы `tb_` для безопасной работы в общей БД).
- **Авторизация**: Кастомная автономная система логин/пароль через таблицу `tb_profiles` (без встроенного Supabase Auth/Email).
- **Telegram Уведомления**: Интеграция с Telegram Bot API для отправки анонимных отчетов в супергруппу компании.

---

## 🗄 2. Структура Таблиц Базы Данных (Supabase SQL)

### `tb_profiles` — Профили пользователей
- `id` (UUID PRIMARY KEY)
- `username` (TEXT UNIQUE) — Логин для входа
- `password` (TEXT) — Пароль
- `name` (TEXT) — Имя и Фамилия
- `avatar` (TEXT) — Эмодзи-аватар
- `color` (TEXT) — Цвет профиля
- `role` (TEXT: `employee` | `manager` | `admin`)
- `is_admin` (BOOLEAN)

### `tb_tasks` — Задачи
- `id` (UUID PRIMARY KEY)
- `task_number` (SERIAL) — Человеческий порядковый номер (`№1`, `№2`...)
- `title` (TEXT)
- `description` (TEXT)
- `status` (`new` | `in_progress` | `stopped` | `done`)
- `progress` (INTEGER: 0, 25, 50, 75, 100)
- `priority` (`low` | `medium` | `high` | `critical`)
- `created_by` (UUID) — Создатель
- `assigned_to` (UUID) — Первичный исполнитель
- `assignees` (TEXT[]) — Массив всех исполнителей
- `responsible_id` (UUID) — Ответственное лицо за прогресс и статус
- `tags` (TEXT[]) — Теги задачи
- `stop_reason` (TEXT)
- `deadline` (DATE)
- `pinned` (BOOLEAN)

### `tb_comments` — Комментарии к задачам
### `tb_task_history` — История изменений
### `tb_notifications` — Внутренние уведомления

## 👥 3. Роли и Права Доступа

1. **👨‍💻 Обычный сотрудник (`employee`)**:
   - Видит задачи, где он является создателем, одним из исполнителей или ответственным.
   - 🔒 **Ограничение прав**: Может оставлять комментарии и общаться в чате задачи. Менять статус и проценты прогресса разрешено **только если сотрудник назначен Ответственным лицо (`responsible_id`)**.
2. **🧑‍💼 Руководитель (`manager`)**:
   - Видит все задачи всех сотрудников компании и полную аналитику.
   - Имеет доступ к модальному окну **`👥 Команда`** и фильтрации задач по сотрудникам.
3. **👑 Администратор (`admin`)**:
   - Полный доступ к системе + доступ к **`⚙️ Панели администратора`**.
   - Регистрация новых сотрудников, управление ролями, удаление пользователей.

## 📢 4. Telegram Уведомления и Конфиденциальность

- **Группа Telegram**: Супергруппа компании (`-1004498774399`).
- **🔒 Политика Конфиденциальности**:
  - Названия и описание задач **НЕ публикуются в Telegram-группе**, чтобы другие сотрудники не читали чужие конфиденциальные поручения.
- **Форматы уведомлений**:
  - 📌 **Новая задача**: `📌 Новая задача №1`, создатель, исполнители, ответственный.
  - 💬 **Комментарий**: `💬 Новый комментарий к задаче №1`, автор.
  - ✅ **Завершение**: `✅ Задача №1 была завершена!`, ответственный, время выполнения (`⏱ X ч Y мин`).

## 🏷 5. Особенности UI и Фильтров

- **Человеческая нумерация**: Задачи отображаются с номерами `№1`, `№2`, `№3` на карточках, в модальных окнах и в Telegram.
- **Теги**:
  - Главный тег выводится в правом верхнем углу карточки напротив приоритета.
  - Список тегов отображается под описанием.
  - Выпадающий фильтр `🏷️ Все теги` над Канбан-доской.
- **Мобильная версия**:
  - Нижняя навигационная панель (`mobile-bottom-nav`).
  - Быстрые плашки колонок доски (`mobile-board-pills`).
  - Плавающая кнопка `+ Задача` (`mobile-fab-btn`).
